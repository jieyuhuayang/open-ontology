# Feature: Search（本体搜索）

**关联 PRD**: [docs/prd/0_1_0（MVP）/本体管理平台（Ontology Manager） PRD.md §搜索功能]
**优先级**: P0
**所属版本**: v0.1.0
**状态**: Draft

---

## 用户故事

作为 **本体管理员或业务分析师**，
我希望 **能够通过关键词快速搜索本体中的对象类型、链接类型和属性**，
以便 **在大型本体中快速定位所需的资源，无需逐页翻阅**。

---

## 验收标准

- **AC1**: 全局搜索框在所有页面可见（导航栏）
- **AC2**: 搜索覆盖范围：对象类型（API name + 显示名）、链接类型（API name + 显示名）、属性（API name + 显示名）
- **AC3**: 搜索结果按资源类型分组展示，每组最多显示 5 条，提供"查看全部"入口
- **AC4**: 点击搜索结果，跳转至对应资源的详情页
- **AC5**: 搜索延迟不超过 300ms（含前端防抖 200ms + 后端响应 < 100ms）
- **AC6**: 输入至少 1 个字符时触发搜索；空输入时不发起请求
- **AC7**: 无结果时显示友好的空状态提示

---

## 边界情况

- **不支持**：模糊匹配 / 拼音搜索（MVP 使用 PostgreSQL `ILIKE` 前缀匹配或 `tsvector` 全文搜索）
- **不支持**：按属性值搜索对象实例（这是 Object Set 功能，超出 Ontology Manager 范围）
- **不支持**：搜索历史记录（延后到 v0.2.0）
- 搜索结果最多返回 50 条（总量），防止性能问题

---

## 非功能要求

- **性能**: 后端搜索接口 P95 响应时间 < 100ms（数据量 < 10,000 条资源）
- **可用性**: 搜索中显示 loading 状态；搜索失败时显示错误提示（不崩溃）

---

## 技术方向

使用 PostgreSQL `tsvector` + `tsquery` 实现全文搜索，或使用 `ILIKE '%keyword%'` 简单实现（MVP 可从简单方案开始）。

参见架构决策文档了解搜索引擎选型（是否引入 ElasticSearch 等外部依赖）。
